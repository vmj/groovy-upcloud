Package: fi.linuxbox.upcloud.api

[Groovy] Class UpCloud

    • fi.linuxbox.upcloud.api.UpCloud

    • Top-level APIs.

      This class provides following APIs:

      • getting account information
      • listing prices, zones, timezones, plans, and server sizes
      • listing and creating servers, IP addresses, tags, and storages

        • def account(def... args)

          Fetch UpCloud API account information for the session.

          A 200 OK response will include an instance of Account in the account property. 

          
     upcloud.account { resp, err ->
         assert resp?.account instanceof Account
     }
          Parameters:
          args - Request callbacks for the GET /account call.
          Returns:
          Whatever is returned by the AbstractSession for starting an asynchronous request.
          See Also:
          UpCloud API docs for GET /account

        • def create(Resource resource, def... args)

          Create a resource.

          For Server creation, a 202 Accepted response will include an instance of Server in the server property. The Server.state property will be maintenance since it will take awhile to create the server.

          For Storage creation, a 201 Created response will include an instance of Storage in the storage property.

          For IpAddress creation, a 201 Created response will include an instance of IpAddress in the ipAddress property.

          For Tag creation, a 200 OK response will include an instance of Tag in the tag property.

          Parameters:
          resource - Resource to create.
          args - Request callbacks for the POST call.
          Returns:
          Whatever is returned by the AbstractSession for starting an asynchronous request.
          See Also:
          UpCloud API docs for POST /server
          UpCloud API docs for POST /storage
          UpCloud API docs for POST /ip_address
          UpCloud API docs for POST /tag

        • def storages(def... args)

          Fetch a list of all storages.

          A 200 OK response will include a list of Storage instances in the storages property. 

          
     upcloud.storages { resp, err ->
         assert resp?.storages instanceof List
         assert resp.storages.every { it instanceof Storage }
     }

          A keyword argument type can be used to narrow down the list. The type can be access type (public or private), or storage type (normal, backup, cdrom, or template), or favorite. 

          
     upcloud.storages type: "template" { resp, err ->
     }

          Only the storages' most relevant information is returned by this operation. Further details on individual storage can be requested with the Storage.load API.

          Parameters:
          args.type - Optional filter for storages: public, private, normal, backup, cdrom, template, or favorite.
          args - Request callbacks for the GET /storage call.
          Returns:
          Whatever is returned by the AbstractSession for starting an asynchronous request.
          See Also:
          UpCloud API docs for GET /storage
